✅ Resumo da Configuração do OpenClaw

Gateway ativo WhatsApp conectado Modelo respondendo

Configurei o OpenClaw para funcionar no WhatsApp com modelo local via Ollama, corrigindo o timeout interno do agente.

Ajustes realizados

  • Serviço do Gateway mantido no systemd --user.
  • Workspace leve definido em ~/.openclaw/wa-lite.
  • Prompt do sistema simplificado em português do Brasil.
  • Perfil mínimo de ferramentas para reduzir sobrecarga.
  • Contexto do agente reduzido para operação mais leve.
  • idleTimeoutSeconds ajustado para evitar bloqueio por inatividade.
  • Modelo principal ajustado para a rota compatível localopenai/qwen2.5:0.5b.
  • Endpoint compatível do Ollama usado em http://127.0.0.1:11434/v1.
  • Canal do WhatsApp mantido em estado conectado e pronto para testes.

Causa do problema

  • O conector nativo anterior do OpenClaw com Ollama entrava em timeout no fluxo embutido.
  • A solução foi usar a rota compatível OpenAI do Ollama e simplificar o ambiente do agente.

Resultado validado

  • O agente voltou a responder sem o erro de timeout.
  • Teste local concluído com resposta em cerca de 18,6 segundos.
  • Status final confirmado: Gateway reachable e WhatsApp connected.
